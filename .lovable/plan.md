

# Панель управления задачами — новый раздел `/tasks`

## Идея

Сейчас задачи живут только внутри проектов (в правом сайдбаре досье). Нужен отдельный глобальный хаб задач — вход через верхнюю навигацию Workspace. Менеджер видит **свои** задачи по всем проектам/сделкам, руководитель — задачи **всей команды** с возможностью эскалаций и переназначений.

---

## Роли

- **Менеджер**: видит только свои задачи, может менять статус, отмечать выполнение
- **Руководитель**: видит задачи всех менеджеров, может переназначать, видит эскалации, сводку по загрузке команды

## Технический план

### 1. Мок-данные (`src/data/mockTasks.ts`)

Агрегировать задачи из всех проектов в единый массив с полями: `projectId`, `projectTitle`, `dealId`, `dealTitle` + все существующие поля `ProjectTask`. Добавить 20-25 задач по 3-4 проектам для демо.

### 2. Новая страница (`src/pages/TasksPage.tsx`)

Роут `/tasks` в `App.tsx`. Страница с тем же header-стилем что и WorkspacePage.

### 3. Компонент `TasksHub` (`src/components/tasks/TasksHub.tsx`)

- Состояние: `viewMode` (list | kanban | calendar | focus)
- Фильтры: проект, сделка, исполнитель, статус, приоритет, блокеры
- Сводка наверху: всего задач / активных / блокеров / просроченных

### 4. Вьюшки (отдельные компоненты):

- **`TasksListView.tsx`** — таблица с сортировкой и группировкой (по проектам/сделкам/исполнителям). Стиль как `DealsTableView`
- **`TasksKanbanView.tsx`** — 3 колонки (active/pending/completed), drag-and-drop карточки. Стиль как `DealsKanbanView`
- **`TasksCalendarView.tsx`** — недельный/месячный вид, задачи на шкале дедлайнов
- **`TasksFocusView.tsx`** — AI-приоритизация: блокеры сверху, потом urgent, потом остальное. Каждая карточка с пояснением «почему важно» и кнопками действий

### 5. Навигация

Добавить таб «Задачи» в хедер WorkspacePage (рядом с существующими табами).

### 6. Мини-JARVIS

Floating кнопка внизу справа — при клике открывается мини-чат для быстрых вопросов по задачам.

## Идеи из документа для доработки основного продукта

**Что хорошо сейчас:**
- Трёхколоночный layout, коллапсирующиеся панели
- Двухуровневая иерархия проект → сделки
- Мини-таблица сделок с переходом на отдельный экран
- Quick Actions Bar

**Что стоит докрутить (не в этой задаче, но на будущее):**
- **Проактивные карточки JARVIS** — JARVIS пишет рекомендации прямо в досье без запроса (сценарий 5 из документа)
- **Генеративный UI в чате** — JARVIS возвращает не текст, а виджеты с action-кнопками (сценарий 4)
- **Health Score** на карточках сделок и проектов (цветовое кодирование 🟢🟡🔴)
- **Forcing Functions** — блокировка перехода между стадиями без выполнения условий
- **Утренний брифинг** — кнопка [☀ Брифинг] генерирует персональный дайджест
- **Экран руководителя** — дашборд с pipeline по всем менеджерам и эскалациями
